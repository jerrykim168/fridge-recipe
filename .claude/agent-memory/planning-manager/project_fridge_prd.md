'냉장고를 부탁해' 프로젝트는 냉장고 사진을 업로드하면 AI 비전 모델이 재료를 인식하고, 별도의 LLM이 레시피를 생성해주는 웹앱이다. 2026-07-01에 PRD를 3단계로 나누어 작성 완료함:
- `PRD_step1.md` — 이미지 업로드 및 AI 비전 모델 기반 재료 인식 (재시도/수동 수정 UX 포함)
- `PRD_step2.md` — 재료 목록을 입력받아 LLM으로 레시피 생성 (요리명/재료/조리순서/예상시간). 조건 커스터마이징(매운맛 제외, 인분수 등)은 요구사항 미확정으로 Out of Scope 처리
- `PRD_step3.md` — **레시피 저장 기능** (v1.1). 사용자가 2단계 결과 레시피를 저장/조회/삭제하는 기능. 사용자 인증 필요 여부는 Open Question으로 남기고, 계정 없이 브라우저 로컬 저장(localStorage 등)을 기본 가정으로 제시.
  - 주의: 최초 작성 시(v1.0) "1·2단계를 하나의 엔드투엔드 웹앱으로 통합"이라는 내용으로 잘못 작성했었으나, 코디네이터의 정정 지시(같은 날)에 따라 "레시피 저장 기능"으로 전면 교체함. 이 프로젝트의 3단계는 통합 웹앱이 아니라 저장 기능이라는 점에 유의.

**Why:** 사용자가 처음부터 새로 기획을 요청했고, 프로젝트 디렉토리가 완전히 비어있는 신규 프로젝트였음(코드 없음, PRD 없음). 3단계 범위는 최초 지시에 명시되지 않아 기획관리자가 "통합 웹앱"으로 임의 해석했었으나, 이후 정정을 받아 "레시피 저장 기능"으로 확정됨.

**How to apply:** 향후 이 프로젝트의 구현/설계 논의 시 이 3개 PRD 파일을 기준 문서로 참조. PRD_step3.md는 저장 기능 관련 문서이며, 1·2단계 통합/전체 UI 흐름에 대한 요구사항은 아직 어느 PRD에도 정의되어 있지 않다는 점을 인지할 것. 만약 사용자가 통합 웹앱에 대한 요구사항을 추가로 준다면 새 PRD(예: PRD_step4.md) 작성을 검토.
